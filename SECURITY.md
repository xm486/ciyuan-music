# 安全政策

## 支持范围

当前开源版主要面向学习、自部署和二次开发。

## 报告安全问题

如果你发现安全问题，请不要直接在公开 Issue 中披露可利用细节。建议通过以下方式联系维护者：

- GitHub Security Advisory
- 私信或邮件联系项目维护者

报告时建议包含：

- 漏洞影响范围
- 复现步骤
- 相关请求或日志
- 建议修复方案

## 部署安全建议

- 修改后台默认账号和密码
- 使用强密码
- 不要公开 `db_config.php`
- 开启 HTTPS
- 限制后台访问路径或 IP
- 定期备份数据库
- 不要上传真实用户数据到公开仓库

## 已知限制

本项目为轻量 PHP 实现，适合个人站点和学习使用。高并发、强安全场景建议自行加固。
